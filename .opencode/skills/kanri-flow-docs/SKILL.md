## ドキュメント生成スキル

### 概要
プロジェクトのドキュメントを効率的に生成・管理します。

### ドキュメント种类

#### 1. 開発者ドキュメント
- **コード構造**: ファイル構成、モジュール関係
- **API仕様**: 関数、変数、クラスの説明
- **開発環境**: セットアップ手順、ビルド方法

#### 2. ユーザーマニュアル
- **操作手順**: 各機能の使い方
- **FAQ**: よくある質問と回答
- **トラブルシューティング**: 問題解決方法

#### 3. デザインドキュメント
- **デザインシステム**: カラーパレット、フォント、コンポーネント
- **UI仕様**: 画面遷移、ユーザーインタラクション

### ドキュメントテンプレート

#### 開発者ドキュメント
```markdown
# {機能名} 仕様書

## 概要
{機能の説明}

## 使い方
{使用方法}

## パラメータ
| パラメータ | 型 | 説明 | 必須 |
|------------|-----|------|------|
| {param1} | {type} | {説明} | {Yes/No} |

## 戻り値
{戻り値の説明}

## 使用例
```
{コード例}
```

## 注意事項
{注意点}
```

#### ユーザーマニュアル
```markdown
# {機能名} マニュアル

## 概要
{機能の説明}

## 操作手順
1. {ステップ1}
2. {ステップ2}
3. {ステップ3}

## 画面説明
{画面の各要素の説明}

## よくある質問
### Q: {質問}
A: {回答}

## トラブルシューティング
### 問題: {問題の説明}
解決策: {解決策}
```

### ドキュメント管理

#### ファイル構成
```
docs/
├── developer/     # 開発者ドキュメント
├── user/          # ユーザーマニュアル
├── design/        # デザインドキュメント
└── api/           # API仕様
```

#### 更新ルール
- 機能追加時にドキュメントを更新
- 定期的にドキュメントを見直す
- チーム全員でドキュメントを共有

### 使用時のポイント
- ドキュメントは正確で分かりやすく
- 定期的に更新を怠らない
- チーム全員がアクセスできる場所に保管
