## プロジェクト管理スキル

### 概要
開発タスクの管理、進捗追跡、リリース計画を効率化します。

### タスク管理

#### タスク分类
- **高優先度**: 機能追加、バグ修正、セキュリティ対応
- **中優先度**: デザイン改善、パフォーマンス最適化
- **低優先度**: ドキュメント作成、リファクタリング

#### タスク記録テンプレート
```markdown
## タスク: {タスク名}
- **ステータス**: 未着手/進行中/完了
- **優先度**: 高/中/低
- **担当**: {担当者名}
- **期限**: {YYYY-MM-DD}
- **進捗**: {進捗率}%
- **備考**: {詳細説明}
```

### 進捗追跡

#### 進捗報告テンプレート
```markdown
## 進捗報告 ({日付})
### 完了タスク
- [ ] タスク1
- [ ] タスク2

### 進行中タスク
- [ ] タスク3 (進捗: 50%)

### 予定タスク
- [ ] タスク4
- [ ] タスク5

### 課題・問題点
- 課題1: {詳細}
- 課題2: {詳細}
```

### リリース計画

#### リリースチェックリスト
- [ ] 全ページの動作確認完了
- [ ] デザインの一貫性確認完了
- [ ] リンク切れ確認完了
- [ ] レスポンシブ確認完了
- [ ] ドキュメント更新完了
- [ ] GitHub Pagesへのデプロイ完了

### 使用時のポイント
- 定期的に進捗を更新
- 課題は早めにチームに共有
- リリース前には必ずチェックリストを確認
